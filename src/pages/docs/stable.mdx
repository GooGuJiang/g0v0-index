# stable 客户端教程

本文指引你在传统 osu!stable 客户端中完成连接与常见问题排查。

<Admonition type="tip" title="阅读顺序">
建议先阅读 **快速开始** 与 **lazer 客户端教程** 了解总体结构与差异。
</Admonition>

## 一、下载与准备

1. 官方渠道获取最新 osu!stable 安装程序。
2. 安装完成后先正常启动一次以生成 `osu!.cfg` 等配置文件。
3. 关闭客户端，准备执行服务器切换操作。

## 二、连接方式

### 方式 1：启动器 / 切换器（推荐）

使用我们提供或社区维护的图形切换器：

1. 打开工具，选择目标服务 `g0v0`。
2. 点击“一键切换”或“应用”。
3. 再次启动 osu!stable 并登录。

### 方式 2：Hosts / DNS 手动配置

<Admonition type="warning" title="风险提示">
手动修改 hosts 可能导致其它域名解析异常，操作前请备份原文件。优先使用图形工具。
</Admonition>

编辑系统 hosts（Windows 可使用管理员权限记事本）追加示例：

```
# 示例，占位用，实际请依据公告提供的解析
203.0.113.55 osu.ppy.sh
```

保存后清理 DNS 缓存 (Windows PowerShell)：`ipconfig /flushdns`

重启客户端验证是否生效。

### 方式 3：代理层 (可选)

使用支持域名重写的本地代理（如 Clash）添加一条规则，将官方域名重定向到目标地址。

## 三、账号与登录

- 第一次登录使用你在本站注册的账号与密码。
- 出现多次失败请确认切换是否生效（可尝试打开官方网页测试）。
- 长期保持在线建议开启“记住密码”。

## 四、成绩与数据

| 项目 | 说明 |
| ---- | ---- |
| 排行榜 | 拉取延迟通常 < 2s，如超时可刷新 |
| 提交失败 | 网络或校验异常，客户端会重试 |
| 改名 | 支持多次改名；改名后历史分数关联保留 |
| PP 刷新 | 异步任务队列，稍有延迟属正常 |

## 五、常见问题 (FAQ)

**Q: 登录显示 Ban？**  
A: 请先确认无作弊类程序；若确认误判提交申诉日志。  

**Q: 谱面加载缓慢？**  
A: 可开启社区镜像下载功能或手动导入本地谱面包。  

**Q: 无法提交成绩？**  
A: 检查 hosts/代理是否还在；关闭可能拦截流量的软件；查看 `update.log`。  

## 六、回退到官方

1. 使用切换器点“恢复官方” 或删除新增的 hosts 记录。
2. 刷新 DNS：`ipconfig /flushdns`。
3. 重启客户端确认恢复。
